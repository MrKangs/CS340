Welcome to the DOGE-X platform database GUI.

Select an item in the navigation bar to view the corresponding database entity.

NOTE: The UI for both the fiatWallet and dogecoinWallet entities DO NOT directly allow you to CREATE a new record. This is because every wallet must have a corresponding user account per our project design. To eliminate the possibility of database integrity errors, we are automatically executing CREATE requests for both a new fiatWallet and dogecoinWallet whenever a new userAccount is created. Similarly, whenever a userAccount, fiatWallet, or dogecoinWallet record is deleted, we are automatically executing DELETE requests for the two other corresponding entity records. Also, pressing each entity's DELETE button successfully queries the MySQL database and delete's the corresponding record. However, you need to refresh/reload the browser in order to see the change reflected in the table. Unfortunately, we were not able to figure out how to reload the page automatically. It appears that the GET request to reload the table sometimes occurs before the POST request to delete the record, so the table doesn't always automatically update; in these instances, manually reloading the page is necessary to view the change to the database.
